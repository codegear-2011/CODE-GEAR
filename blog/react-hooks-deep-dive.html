React Hooks Deep Dive

Understand React Hooks like useState, useEffect, and custom hooks in depth.

By CODE GEAR · September 30, 2025

Hooks changed the way we write React. They allow you to use state and other React features without writing a class. This article covers the most important hooks (useState, useEffect) and shows how to compose logic into reusable custom hooks.

Why Hooks?

Hooks make it easier to share logic between components, reduce boilerplate, and help keep components more readable. They also encourage functional components and cleaner separation of concerns.

useState — Local state in a function

`useState` returns a stateful value and a function to update it. It's great for component-local state like form inputs, toggles, counters, etc. 


// Example: simple counter component
import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(c => c + 1)}>Increment</button>
    </div>
  );
}

useEffect — side effects & lifecycle

`useEffect` runs side-effects (data fetching, subscriptions, timers). You can control when it runs with a dependency array. 


// Example: timer with cleanup
import React, { useState, useEffect } from 'react';

function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const id = setInterval(() => setSeconds(s => s + 1), 1000);
    return () => clearInterval(id); // cleanup on unmount
  }, []); // empty array -> run once after mount

  return <div>Seconds: {seconds}</div>;
}

Common useEffect patterns

  • Run once on mount: `useEffect(..., [])`
  • Watch values: `useEffect(..., [value])`
  • Cleanup: return a function to unsubscribe / clear timers

Custom Hooks — extract reusable logic

Custom hooks are just JS functions that use hooks. They let you share behavior across components. 


// Example: useFetch custom hook
import { useState, useEffect } from 'react';

function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    let mounted = true;
    setLoading(true);
    fetch(url)
      .then(r => r.json())
      .then(json => { if (mounted) setData(json); })
      .catch(err => { if (mounted) setError(err); })
      .finally(() => { if (mounted) setLoading(false); });

    return () => { mounted = false; };
  }, [url]);

  return { data, loading, error };
}

// Usage in a component:
function Posts() {
  const { data, loading, error } = useFetch('/api/posts');
  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error</div>;
  return <ul>{data.map(p => <li key={p.id}>{p.title}</li>)}</ul>;
}

Rules of Hooks (short)

  • Only call hooks at the top level (no loops, conditionals, nested functions).
  • Only call hooks from React function components or custom hooks.
  • Name custom hooks with the `use` prefix (e.g. `useAuth`).

Advanced Tips

useRef is great for keeping mutable values across renders (e.g., storing timers or previous values). useMemo and useCallback help optimize expensive calculations and stable callbacks — but only use them when necessary. 


// Example: useRef to focus input
import React, { useRef } from 'react';

function FocusInput() {
  const ref = useRef(null);
  return (
    <div>
      <input ref={ref} placeholder="Type..." />
      <button onClick={() => ref.current && ref.current.focus()}>Focus</button>
    </div>
  );
}

When to create a custom hook?

Create a custom hook when you find repeating logic across multiple components (data fetching, form handling, subscriptions). It improves readability and testability.

Conclusion

Hooks are the modern way to build React apps. Start with `useState` and `useEffect`, then refactor repeated logic into well-named custom hooks. Keep the Rules of Hooks in mind, and favor small, focused hooks for clarity.
